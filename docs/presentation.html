Motivation

The Clinical Problem

🏥 Manual Interpretation is Slow

Radiologists manually trace vertebrae, IVDs, and the spinal canal across dozens of MRI slices — a time-consuming, subjective process prone to inter-reader variability.

📊 Multi-modal Data Underused

Clinical reports, demographics, and MRI images are rarely fused systematically, leaving valuable diagnostic signal untapped in existing AI solutions.

🎯 What We Need

  • Pixel-level segmentation of vertebrae & discs
  • Automatic disease detection & severity grading
  • Explainable, trustworthy AI predictions
  • Integration of MRI + text reports + demographics
  • Clinician-ready PDF output

💡 ATM-Net++ Solution

A production-grade clinical AI platform that takes MRI + radiology text + patient demographics and returns segmentation maps, disease labels, severity scores, and structured reports — all with Grad-CAM explainability.

Novel Contributions

What Makes ATM-Net++ New

① ATPG — Anatomy-Text Prompt Generation

Generates anatomy-aware text prompts guided by image features. Bridges the visual and language spaces by conditioning BERT token attention on spatial encoder activations.

Cross-modal conditioning16 learnable prompts

② HASF — Hierarchical Anatomy-Aware Semantic Fusion

Fuses image, text and demographic embeddings through multi-head cross-attention at multiple hierarchical scales, preserving both global context and local anatomy.

8 attention heads4 transformer layers

③ CCAE — Cross-modal Context-Aware Enhancement

FiLM-style modulation: injects fused multimodal context into decoder feature maps via learned scale and shift parameters, refining segmentation with semantic knowledge.

FiLM conditioningDecoder injection

④ Joint Contrastive Image-Text Alignment

NT-Xent contrastive loss aligns image and text feature projections in a shared 256-dim embedding space, improving cross-modal grounding without extra labeled pairs.

NT-Xent loss256-dim shared space
Implementation

Model Components

🖼 Image Encoder

  • Swin UNETR backbone
  • Residual blocks + attention gates
  • Deep supervision (3 levels)
  • Feature size: 48 → 768-dim global
  • Lightweight U-Net fallback (no MONAI)
  • Input: (B, 1, 512, 512)

📝 Text Encoder

  • Bio-ClinicalBERT (Alsentzer 2019)
  • First 6 layers frozen
  • Fine-tuned head
  • CLS embedding: 768-dim
  • Max sequence: 512 tokens
  • Token embeddings for ATPG

👤 Demographic Encoder

  • 8-dim input vector
  • 3-layer MLP: 64→128→256
  • Dropout 0.2
  • Fields: age, sex, BMI,
    height, weight, symptoms
  • Output: 256-dim

🧩 Multi-Task Heads

  • Disease classification (7-class)
  • Severity: 3-class + regression
  • Level localization: 8 IVDs, multi-label
  • IVD pathology + Pfirrmann grade
  • All conditioned on fused 512-dim

📄 Report Generator

  • Neural report head (NeuralReportHead)
  • Template + neural hybrid
  • Findings, impression, recommendation
  • ReportLab PDF rendering
  • Radiologist-style structure

🔍 Explainability

  • Grad-CAM (Selvaraju 2017)
  • Attention Rollout (Abnar 2020)
  • Segmentation overlay (α=0.4)
  • Disease localization heatmaps
  • Base64 encoding for API
Training

Training Configuration & Losses

⚙️ Optimizer & Schedule

  • AdamW · lr=1e-4 · wd=1e-5 · β=(0.9, 0.999)
  • Cosine LR + 10-epoch linear warmup
  • Min LR: 1e-6
  • 200 epochs · batch size 4 · grad accum ×4
  • Mixed precision (FP16 AMP)
  • Early stopping · patience=30 on val Dice

🔁 Augmentation

  • Random rotation ±15°
  • Horizontal/vertical flip (p=0.5)
  • Elastic deformation (α=100, σ=10)
  • Intensity shift ±0.1, scale ±0.1
  • Gaussian noise σ=0.01
  • Random crop 480×480, zoom 0.9–1.1×

Evaluation

Metrics & Evaluation Protocol

📐 Segmentation Metrics

  • Dice Score — per-class & mean (target >0.90)
  • Jaccard / IoU — intersection over union
  • HD95 — 95th percentile Hausdorff Distance
  • ASD — Average Surface Distance
  • Precision, Recall, F1

🏥 Classification Metrics

  • Disease classification accuracy
  • Macro F1 score across 7 classes
  • Per-class AUC
  • Severity estimation MAE + accuracy

🔬 Inference Features (TTA)

  • Test-Time Augmentation (3 flip combinations)
  • Threshold: 0.5 on softmax probabilities
  • Batch size 1 for production inference

💾 Outputs Saved

  • Segmentation mask (.npy/.mha)
  • Overlay PNG
  • Grad-CAM heatmap PNG
  • Structured JSON results
  • Clinical PDF report

Getting Started

Quick Start

① Install dependencies

# Python
pip install -r requirements.txt

# Frontend
cd frontend && npm install && cd ..

② Configure & set up data

copy .env.example .env
# Edit .env: SECRET_KEY, DB settings

python scripts/setup_data.py \
  --source "C:\project\Spine Segmentation\10159290"

③ Launch full stack

docker-compose up --build

# Frontend   → http://localhost:3000
# API docs   → http://localhost:8000/docs
# Health     → http://localhost:8000/health

④ Train or run inference

# Train
python training/train.py \
  --config configs/base_config.yaml

# Inference
python scripts/run_inference.py \
  --image data/.../100_t2.mha \
  --checkpoint checkpoints/best.pth \
  --save-report outputs/report.pdf
